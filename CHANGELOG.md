# 변경 내역 (Changelog)

## 리팩토링 1 + Load 속도 개선

- **Load 느려짐 수정** — 통계(가우시안 수) 계산이 패널이 닫혀 있어도 매 버퍼 갱신마다 전체 버퍼를 스캔하던 것을, **통계 패널이 열렸을 때만** 계산하도록 게이트. delta 스트리밍 중 전체 스캔 ~20회 제거.
- **중복 제거** — `src/lib/gaussianEdit.ts`(hexToRgb·viewOf·readCov6/writeCov6·avgColorHex)로 색 파싱·공분산 f16 read/write·평균색 중복 통합.
- **훅 분리** — 키보드 단축키를 `src/hooks/useKeyboardShortcuts.ts`로.

## 실시간 폴링 (#15)

- delta 로드 시 상단 `라이브` 토글 — 3초마다 manifest를 폴링해 새 프레임을 버퍼에 append(라이브 스캔 반영). 마지막 프레임을 보고 있으면 최신으로 자동 따라감.

## 그룹 (#1)

- 상단 `그룹` 패널 — **선택을 그룹으로 저장** 후 **재선택 / 숨김·보기 / 색 지정(스와치) / 해제**. 그룹 숨김은 display 오버레이에 통합(피킹에서도 제외). 로드/reset 시 초기화. (통째 이동·가우시안 삭제는 재선택 후 핸들·Delete로)

## 필터 선택 (#13)

- 상단 `필터` 토글 패널 — **색 유사도**(대상 색 + 허용 오차) 및 **불투명도 범위**로 대량 선택. "기존 선택에 추가" 옵션, "선택색"으로 현재 선택의 평균색 추출. (위치범위·voxel은 추후)

## 숫자 입력 이동/회전 (#25)

- 선택 이동 step·회전 각도를 슬라이더 옆 **숫자 입력**으로 직접 지정(슬라이더 범위를 넘는 값도 가능).

## 선택 반전 / 확장 (#24)

- 선택 패널에 **선택 반전**(보이는 것 중 미선택 전체) + **확장**(선택 박스 안을 채워 넓힘).

## 비균등 스케일 (#23)

- 선택 스케일에 **축별(X/Y/Z) − / ＋** 추가(기존 균등 스케일 유지). 위치+공분산 함께 변환.

## 선택만 .ply 내보내기 (#22)

- 선택 패널에 `선택만 .ply 내보내기` — 전체 대신 선택한 가우시안만 저장.

## Redo + 키보드 단축키 (#21)

- **Redo** 추가(undo/redo 스택). 새 편집 시 redo 스택은 무효화. 툴바에 `redo` 버튼.
- **단축키** — Ctrl/Cmd+Z 실행취소, Ctrl+Shift+Z(·Ctrl+Y) 다시실행, Delete/Backspace 선택 삭제, Esc 선택 해제. (입력창 포커스 중엔 무시)

## 선택 회전 각도 슬라이더

- 선택 회전이 고정 15°였던 걸, 이동 step처럼 **각도 슬라이더(1~90°)** 로 설정 후 X/Y/Z ⟲/⟳ 적용하도록 변경.

## 회전 속도 기준을 "콘텐츠 근접도"로

- 회전 속도 배율을 카메라↔타깃 거리가 아니라 **카메라에서 가장 가까운 가우시안까지 거리**(서브샘플 2000개)로 계산. 줌·플라이·텔레포트 등 **어떻게 가까워졌든** 실제로 콘텐츠에 가까우면 회전이 느려짐(타깃이 멀리 있어도 과민해지지 않음).

## 확대 시 회전 과민 완화

- 줌 거리에 비례해 OrbitControls 회전 속도 자동 조절(`AdaptiveRotateSpeed`). 가까이 갈수록 회전을 느리게(최소 0.1배) 해 줌인 상태에서 미세 조작 가능. 기본/멀리선 그대로(1.0).

## 씬 전체 회전 (기울기 보정) + 카메라 관성 제거

- ⚙ 설정에 **씬 회전** 섹션 — 선택과 무관하게 **모든 가우시안을 씬 중심 기준 X/Y/Z 축으로 5°씩** 회전(위치+공분산). 뒤틀려 캡처된 씬을 그리드(수평)에 맞춰 펴기 좋음. undo 가능.
- **카메라 관성 제거** — OrbitControls `enableDamping={false}`. 놓으면 미끄러지던 관성이 사라져 즉시 멈춤.

## 버그: LOD 조작 시 가우시안 전부 사라짐

- 원인: 버퍼 크기 변경 경로에서 새 `sortedIndex`가 전부 0이라 모든 인스턴스가 "가우시안 0"으로 겹침. 워커 정렬은 카메라 이동 시에만 도는데, LOD는 카메라를 안 움직여서 안 풀림.
- 수정: 크기 변경 시에도 identity 순서(0,1,2,…)를 즉시 채워 모두 보이게. (스트리밍 시 잠깐 깜빡이던 것도 같이 개선)

## 스케일 바 제거

- 불필요해서 화면 스케일 바와 관련 코드(`ScaleBarProbe`·토글) 전부 제거. (측정 기능은 유지)

## 카메라 북마크 (#8) + 회전 링 두껍게

- **카메라 북마크** — ⚙ 설정에서 `현재 시점 북마크 저장` → 목록에서 눌러 복귀, ✕로 삭제. localStorage 영구 저장. (위/정면/측면/대각·원점 프리셋과 함께 #8 완료)
- **회전 링 두께 ↑** — 선택 회전용 초록 링을 두껍게(터치로 잡기 쉽게) + 살짝 키움.

## 사용성: 설정 닫기·선택 해제 + 스케일 바 (#9)

- **고급 설정 패널 상단 ✕** — 스크롤해도 고정(sticky)되는 닫기 버튼. 모바일에서 ⚙가 메뉴 안에 있어도 패널을 바로 닫을 수 있음.
- **명시적 선택 해제** — 선택 패널 헤더에 ✕(선택 해제) 추가. 빈 곳 더블탭에 의존하지 않아도 됨(접힘/모바일에서도 항상 보임).
- **스케일 바 (#9)** — 화면 하단 중앙에 현재 배율의 실측 눈금(├─ N ─┤). 카메라 거리에 따라 자동 갱신, ⚙에서 on/off.

## 로드 입력값 기억 (localStorage)

- 서버 URL(host)과 run·mode·maxFrames를 localStorage에 저장 → 재방문 시 자동 복원, 다시 입력 안 해도 됨. (URL 공유 파라미터가 있으면 그쪽이 우선, private 모드 등 저장 불가 시 무시)

## 태블릿 지원

- 햄버거 접힘 트리거를 **폰 + 터치 태블릿(`pointer: coarse` 이면서 ≤1366px)** 으로 확장. 태블릿에서도 풀 툴바가 여러 줄로 쌓이지 않고 ☰로 깔끔하게 접힘. 마우스 데스크톱·큰 터치 모니터(>1366px)는 풀 툴바 유지. (터치 타깃 확대·반응형 패널 폭은 기존 적용분으로 태블릿도 커버)

## 모바일 햄버거 메뉴 + 타임라인 토글

- **모바일 툴바 접기** — 좁은 화면(≤640px)에선 필수 버튼(Load·undo·reset)만 보이고, 나머지는 반투명 **☰ 토글**로 펼침/접힘. 데스크톱은 그대로(순서·레이아웃 불변, CSS `.menu-only`/`.collapsed`로 처리). 도움말은 모바일 기본 닫힘 유지.
- **타임라인 on/off** — `타임라인` 버튼으로 구간 슬라이더 패널을 켜고 끌 수 있음(delta 로드 시). 끄면 도움말/통계 패널이 하단으로 다시 내려옴.

## 반응형 UI (모바일 대응)

- **패널이 화면을 안 넘침** — 툴바·선택·설정·통계·도움말·측정 패널 너비를 `min(px, calc(100vw - …))`로 잡아 좁은 화면에서 뷰포트 안에 맞춤. 툴바는 좁아지면 줄바꿈 + 높이 상한(42vh)·스크롤로 화면을 다 가리지 않음.
- **터치 타깃 확대** — `@media (pointer: coarse)`에서 버튼/입력 최소 높이·체크박스·슬라이더 thumb를 손가락용으로 키움.
- **좁은 화면 컴팩트화** — `@media (max-width: 640px)`에서 패딩·폰트 축소, 도움말 패널은 모바일에서 기본 닫힘.
- 터치 조작(한 손가락 회전, 핀치 줌/팬, 핸들 드래그)은 기존 pointer 이벤트로 그대로 동작. (viewport meta·user-scalable=no 기존 설정 활용)
- **패널 잘림 방지** — 모든 floating 패널에 위치 오프셋을 감안한 최대 높이(`100dvh` 기준, 모바일 주소창 대응) + 세로 스크롤. 화면보다 길면 잘리지 않고 패널 안에서 스크롤됨(특히 도움말). 가로도 `100vw` 안으로 캡.

## 카메라 원점 이동 + WASD 앞/뒤를 줌과 동일화

- **카메라를 축(원점) 위치로** — ⚙ axes 토글 옆 버튼. 카메라를 월드 원점(축 기즈모 위치, 보통 캡처/기준 원점)에 두고 데이터 중심을 바라봄.
- **WASD 앞/뒤 = 줌(dolly)** — 앞/뒤(W/S·↑↓)는 이제 타깃 쪽으로 dolly(휠 줌과 동일): 거리가 줄어 가까워질수록 느려지고, 멀어질수록 빨라짐(끝까지 가도 과속 안 함). 좌우/상하(A/D·Q/E)만 pan(타깃 같이 이동). 타깃이 콘텐츠에 머물러 줌 후 회전이 튀던 문제도 완화.

## 성능(LOD) + 화면고정 핸들 + 뷰 프리셋

- **렉 줄이기 (LOD)** — ⚙ 설정의 `표시 비율(%)` 슬라이더로 그릴 가우시안을 솎음(매 N번째만 렌더 → 정렬·래스터 부하 감소). **선택·편집·피킹은 전체 버퍼 기준 그대로** 유지. 카메라만 움직일 땐 재계산 없음. (기존 dpr·cull·maxFrames와 함께 사용)
- **핸들 화면 고정 크기** — 선택 이동(주황 구)·회전(초록 링)·측정(파란 구)이 카메라 거리와 무관하게 **항상 같은 픽셀 크기**로 보이도록(매 프레임 거리 비례 스케일).
- **뷰 프리셋** — ⚙ 설정에 위/정면/측면/대각 카메라 프리셋. (촬영 방향 추정 "자동 뷰"·"뷰 맞춤"은 정확도가 낮아 제거)

## PLY 로드 버그 수정 + 구간 .ply 리플레이

- **버그 수정** — 가우시안을 한 번 로드한 뒤 `PLY 열기` 하면 화면이 비던 문제: "Load 시 카메라 유지" 정책 때문에 카메라가 이전 장면을 보고 있어 새 PLY가 화면 밖이었음. PLY 열기는 새 파일이므로 카메라를 그 데이터에 다시 핏하도록 수정(서버 Load는 기존대로 카메라 유지).
- **구간 .ply 리플레이** — 내보낸 `.ply`에 프레임 정보(`ushort frame`, 표준 뷰어는 무시)를 함께 저장 → 다시 `PLY 열기`로 불러오면 **타임라인 슬라이더로 재생/스크럽 가능**. 구간 추출 시 프레임은 0부터 재정렬돼 구간만 리플레이됨. (`packedToPly(buffer, frames?)`, `parsePly`가 `{buffer, frameCum}` 반환)

## WASD / 방향키 카메라 이동

- WASD·방향키로 카메라 플라이 이동(`KeyboardFly`). OrbitControls와 함께 동작하도록 카메라와 타깃을 같이 이동. 속도는 줌 거리에 비례(어느 배율에서도 자연스러움), Shift=3배 빠르게, Q·E(또는 Space)=아래·위(world-up z). 입력창 포커스 시엔 무시, 방향키/스페이스의 페이지 스크롤은 방지.

## 회전 링 핸들 + 실시간 이동/회전

- **회전 링 핸들** — 선택 중심에 카메라를 향하는 초록 링(기즈모). 잡고 돌리면 시점축 기준으로 회전(블렌더 R과 동일; 다른 축은 카메라를 돌린 뒤 회전). ±15° 버튼은 정밀 보정용으로 유지.
- **실시간 이동/회전** — 주황 구(이동)·초록 링(회전) 드래그가 **드래그 중 라이브로 반영**됨. `liveBuffer`로 미리보기, 드래그당 선택분만 다시 써서 큰 버퍼에서도 가볍고, undo는 드래그당 1회. (이동/회전 핸들 인터페이스를 commit-on-release → onStart/onMove/onEnd로 전환)
- mathUtils 공유: 버튼·링·라이브 회전이 모두 `rotateCovariance`로 위치+공분산 동시 변환.

## 재생 속도 자유 조절 (드래그)

- 고정 프리셋(2/5/10/20/30fps)을 없애고, fps 컨트롤을 **좌우 드래그로 자유 조절**(0.2fps/px, 0.5 단위, 0.5–60fps). 조금 드래그하면 조금, 많이 드래그하면 많이 변경. 0.5fps(프레임당 2초) 같은 저속 재생도 가능.

## PLY 불러오기 + 타임라인 구간 UX 개편

- **PLY 열기** — 로컬 `.ply` 파일을 뷰어로 직접 로드(서버 불필요). `parsePly`가 표준 3DGS binary PLY 파싱 → `covarianceFromScaleRotation`+`packSplats`로 패킹(내보내기의 역). f_dc/opacity/scale/rot 활성화 처리, 색만 있는 포인트 클라우드는 red/green/blue + 작은 등방성 스케일로 폴백.
- **타임라인 구간 UX 개편** — 풀폭 슬라이더(화면 가로 전체). 트랙에 구간 시작/끝 **하이라이트 밴드 + 마커 라벨**. 버튼을 `구간 시작`·`구간 끝`·`구간 해제`로 명확화. 하단 풀폭 타임라인과 겹치지 않게 도움말/통계 패널은 위로 이동.
- **검증** — `npm test`에 PLY 왕복(packed→ply→parse, 위치/색 일치) 셀프체크 추가.

## 타임라인 재생 + 구간 .ply 추출

- **재생 / 일시정지** — ▶/⏸ 버튼으로 delta 프레임 자동 진행(구간 [in,out] 내에서 루프). 슬라이더 수동 조작 시 정지.
- **재생 속도** — `{n}fps` 버튼으로 프리셋(2·5·10·20·30fps) 순환.
- **구간 추출** — 현재 프레임을 구간 시작/끝으로 지정 후 `구간 .ply` → 해당 프레임 구간에 추가된 가우시안만 `.ply`로 추출(`packedToPly`에 subarray 전달, alpha 0 제외). *영상 녹화가 아니라 ply 추출 해석.*
